# 캠페인 의사결정 온톨로지

기획 → 실행 → 회고 3단계를 통해, 기존 Marketing-Brain 데이터를 CONTEXT로 활용하여 반자동화된 마케팅 의사결정을 수행하는 시스템.

---

## 플로우

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│  기획.md  │────▶│  실행.md  │────▶│  회고.md  │
│ CONTEXT  │NEXT │  INPUT   │NEXT │  INPUT   │
│ PROMPT   │────▶│ W1~W4    │────▶│ PROMPT   │
│ OUTPUT   │     │ PROMPT   │     │ OUTPUT   │
│ NEXT     │     │ OUTPUT   │     │ ACTION   │
└──────────┘     │ NEXT     │     └────┬─────┘
                 └──────────┘          │ 피드백
                                       ▼
                              ┌─────────────────┐
                              │  [[회고_모음]]   │
                              │  학습 누적       │
                              │  → 다음 캠페인   │
                              │    기획 C6에 반영│
                              └─────────────────┘
```

---

## 단계별 사용법

### 1단계: 기획
1. `_Templates/기획_템플릿.md`를 캠페인 폴더로 복사 → `기획.md`로 이름 변경
2. [CONTEXT] C1~C6 블록에 해당 파일 데이터를 붙여넣기
3. [CONTEXT]+[PROMPT]를 복사하여 Claude에게 질문
4. Claude 응답을 [OUTPUT] 각 섹션에 기록
5. [NEXT]에 실행단계로 넘길 정리본 작성

### 2단계: 실행
1. `_Templates/실행_템플릿.md`를 캠페인 폴더로 복사 → `실행.md`로 이름 변경
2. [INPUT]에 기획의 [NEXT] 붙이기
3. 매주 KPI 실적 테이블 채우기
4. W1~W3 주간 [PROMPT]를 Claude에게 질문 → [결정기록]에 기록
5. W4 완료 후 [NEXT]에 최종 정리본 작성

### 3단계: 회고
1. `_Templates/회고_템플릿.md`를 캠페인 폴더로 복사 → `회고.md`로 이름 변경
2. [INPUT]에 실행의 [NEXT] 붙이기
3. [PROMPT] 4개 질문을 Claude에게 질문 → [OUTPUT]에 기록
4. [ACTION] 피드백 루프 3단계를 순서대로 실행
5. 완료 후 캠페인 폴더 내 학습 메모 남기기

## 캠페인 폴더 규칙

| 규칙 | 예시 |
|------|------|
| 폴더명 | `YYYYMM_제품명` |
| 파일구성 | `기획.md` / `실행.md` / `회고.md` |
| 날짜 형식 | YYYY-MM-DD |

예시: `202604_캠퍼세트/`, `202607_여름바베큐/`
