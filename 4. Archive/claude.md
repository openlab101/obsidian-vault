# 4. Archive

## 폴더 목적

완료된 프로젝트와 더 이상 활성화되지 않은 항목을 보관하는 저장소입니다.

## 들어가는 노트

- 완료된 프로젝트
- 더 이상 관리하지 않는 Areas
- 사용하지 않는 Resources
- 참고용으로만 보관할 과거 자료

## 작업 방식과 규칙

1. **이동 시점**: 프로젝트 완료 또는 영역 비활성화 시
2. **날짜 기록**: 아카이브 날짜를 노트에 표시
3. **검색 가능**: 필요시 검색으로 찾을 수 있도록 유지
4. **정기 정리**: 6개월-1년 주기로 완전 삭제 검토

## Archive 구조

- 연도별 또는 카테고리별로 하위 폴더 생성 가능
- 예: `4. Archive/2025/`, `4. Archive/Projects/`

## 아카이브 노트 표시 예시

```markdown
# [프로젝트명]

> [!info] Archived
> 완료일: YYYY-MM-DD
> 아카이브 사유: 프로젝트 완료

[원래 내용]
```
