# 20. Permanent

## 폴더 목적

여러 출처를 종합하여 생성한 나만의 영구적인 지식과 통찰을 저장하는 지식 발전소의 핵심입니다.

## 들어가는 노트

- 여러 Literature 노트를 종합한 개념
- 나만의 프레임워크나 모델
- 검증된 개인적 통찰
- 원자적(atomic)이고 독립적인 아이디어
- 시간이 지나도 유효한 지식

## 작업 방식과 규칙

1. **원자성(Atomic)**: 하나의 노트에 하나의 명확한 아이디어
2. **내 언어**: 완전히 내 언어로 작성, 외부 출처 의존 최소화
3. **독립성**: 각 노트는 독립적으로 이해 가능
4. **연결성**: 다른 Permanent 노트와 풍부하게 연결
5. **발전성**: 새로운 인사이트가 생기면 노트 업데이트 또는 새 노트 생성
6. **명확한 제목**: 제목만 봐도 내용을 알 수 있게

## Permanent 노트 구조 예시

```markdown
# [명확한 개념 제목]

## 핵심 아이디어
[하나의 명확한 개념을 내 언어로 설명]

## 설명
[더 자세한 내용]

## 예시
구체적인 적용 사례나 예시

## 연결된 개념
- [[관련 개념 1]] - 이 노트와의 관계 설명
- [[관련 개념 2]] - 이 노트와의 관계 설명
- [[반대 개념]] - 대조되는 관점

## 출처
이 아이디어를 형성하는데 기여한 Literature 노트들:
- [[Literature 1]]
- [[Literature 2]]

## 내 적용
실제로 어떻게 활용하고 있는지
```

## Permanent 노트의 특징

- **재사용 가능**: Projects나 Areas에서 반복적으로 참조
- **지식의 핵심**: 시간이 지나도 가치가 유지
- **연결의 허브**: 다른 노트들을 연결하는 중심점
- **사고의 단위**: 나의 사고 체계를 구성하는 기본 블록
