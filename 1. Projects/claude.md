# 1. Projects

## 폴더 목적

명확한 마감일과 완료 조건이 있는 단기 프로젝트를 관리하는 작업 공간입니다.

## 들어가는 노트

- 특정 목표와 마감일이 있는 프로젝트
- 여러 단계의 작업이 필요한 업무
- 완료 후 Archive로 이동할 항목들
- 예시: 공모전 준비, 캠페인 기획, 제품 출시 프로젝트

## 작업 방식과 규칙

1. **프로젝트 생성**: 새 프로젝트 시작 시 `YYYYMMDD_프로젝트명.md` 형식으로 생성
2. **상태 관리**: 프로젝트 노트에 상태 표시 (진행중, 대기, 완료)
3. **완료 처리**: 프로젝트 완료 시 `4. Archive/`로 이동
4. **연결**: 관련된 Zettelkasten 노트나 Resources를 링크로 연결

## 프로젝트 구조 예시

```markdown
# 프로젝트명

## 목표
[명확한 완료 조건]

## 마감일
YYYY-MM-DD

## 진행 상태
- [ ] 작업 1
- [ ] 작업 2
- [x] 작업 3

## 관련 자료
- [[Literature Note]]
- [[Resource]]
```
